# CUMCM 2015B “互联网+”时代的出租车资源配置案例记录

## 适用题型

- 题目只有开放式文字、无统一数据附件；要求“搜集相关数据”或自行建立数据口径。
- 城市交通平台供需匹配、出租车/网约车资源配置、双边市场补贴策略设计。
- 评价 + 政策评估 + 优化机制设计的组合题。

## 本次可复用建模路线

### S0 输入资产预检

- 原题目录只有 `CUMCM-2015-problem B-Chinese.doc`，无 Excel/CSV 附件。
- 可用 `antiword` 提取 `.doc` 题面文本；若没有真实数据附件，不要假装题目给了订单数据。
- 支撑材料仍应按标准结构原地创建：`支撑材料/papper/`、`quest1/`、`quest2/`、`quest3/`、`data/`、`results/`、`tables/`、`contracts/`。

### Q1 供求匹配评价

- 评价对象：区域 × 时段 × 天气的时空单元。
- 指标体系：订单成功率、供需比接近度、平均等待时间、绝对供需缺口、空驶率。
- baseline：等权综合评分。
- 主模型：熵权法确定客观权重 + TOPSIS 计算供求匹配指数。
- 验证：与等权 baseline 的排序相关性、区域/时段热力图、指标权重解释。

常用公式：

```text
R_i = S_i / D_i
H_i = 1 / (1 + |R_i - 1|)
C_i = D_i^- / (D_i^+ + D_i^-)
```

### Q2 补贴效果评估

- “缓解打车难”不要只看订单量；应至少同时看成功率、等待时间、供需比和单均补贴成本。
- baseline：补贴前后均值对比。
- 主模型：含平台、时段、天气控制变量的回归模型，区分乘客端补贴和司机端补贴。
- 关键解释：乘客端补贴会刺激需求；在短缺场景中，若供给没有同步增加，可能不缓解甚至加剧打车难。司机端补贴通常更直接改善有效供给。

示例回归形式：

```text
p_it = beta0 + beta1 Post_t + beta2 passenger_subsidy_it
       + beta3 driver_subsidy_it + beta4 mismatch_it
       + platform FE + period FE + weather FE + error_it
```

### Q3 新平台补贴方案

- 不建议设计固定补贴作为最终方案；应按实时错配程度动态补贴。
- 状态变量：`m_i = max(0, D_i - S_i) / D_i`。
- 候选函数：

```text
s_p,i = clip(a m_i + b, 0, passenger_cap)
s_d,i = clip(c m_i + d, 0, driver_cap)
```

- 目标函数可综合成功率、供需比接近度、等待时间转化得分和成本惩罚。
- baseline：无补贴 + 固定双边补贴。
- 验证：补贴强度 ±30% 敏感性；比较匹配指数、成功率、等待时间、单均成本。

## 数据口径与诚信说明

- 若题目无附件且无法联网获取可靠订单级数据，可构造“可复现实证口径”的估计数据集，但必须在论文中明示：数据是基于城市功能区、时段、天气与供需机制的结构化估计样本，不代表某一城市真实平台订单。
- 所有随机生成数据必须固定随机种子，保存到 `data/`，并把关键数字冻结到 `results/frozen_numbers.json`。
- 摘要和正文必须避免把估计样本写成真实调研数据。

## 论文写作要点

- 题目开放时，数据口径章节要比普通题更详细，说明为什么选择区域/时段/天气维度。
- Q1 重点解释指标含义与权重；Q2 重点解释“补贴对象不同导致效果不同”；Q3 重点论证动态补贴优于固定补贴。
- 结论表达推荐：
  - 哪些区域/时段匹配最差；
  - 司机端补贴和乘客端补贴的方向性差异；
  - 新方案的平均补贴、成功率、等待时间和相对 baseline 改进。

## LaTeX/交付踩坑

- 不要在模板中无必要引入 `tocloft`；部分 MiKTeX 环境缺少该包，目录不需要复杂定制时直接删除即可。
- 图表文件名必须与 `\includegraphics` 完全一致；编译报 `Image inclusion failed` 时先搜索 `.tex` 中图片名，再核对 `results/figures/`。
- 若终端连续编译失败，先读最新日志定位缺失包/缺失图片，不要盲目重复运行 `xelatex`。
- 最终必须用 `pdfinfo 论文.pdf` 核验页数，并用 zip 内容清单核验 PDF、代码、冻结数字、结果表和 README 是否进入压缩包。
