# CUMCM 2015D 众筹筑屋规划方案设计案例

## 触发场景

用户要求完成 2015 年国赛 D 题“众筹筑屋规划方案设计”或类似的房地产户型组合规划、土地增值税核算、投资回报率判断问题时参考本记录。

## 题目资产与预检

典型原始文件：

- `CUMCM-2015-Problem D-Chinese.doc`：题面；
- `附件1 赛题所需的相关数据.doc`：11 种房型、方案 I、核算参数、房型上下限、满意比例；
- `附件2 中华人民共和国土地增值税暂行条例.pdf`：土地增值税四级超率累进税率；
- `附件3 其他相关说明.pdf`：补充政策说明。

`antiword` 可提取 `.doc` 题面和附件 1；`pdftotext -layout` 可提取附件 2/3，但政策 PDF 可能因字体编码导致中文不完整，关键税率表需人工核对：

| 增值额 / 扣除项目金额 | 税率 | 速算扣除系数 |
|---|---:|---:|
| ≤50% | 30% | 0 |
| 50%--100% | 40% | 5% |
| 100%--200% | 50% | 15% |
| >200% | 60% | 35% |

## 数据口径

附件 1 关键数据：

- 土地面积：`102077.6` 平方米；
- 取得土地支付金额：`777179627` 元；
- 与转让房地产有关税金：按收入 `5.65%` 计算；
- 最大容积率：`2.28`；
- 方案 I 总套数：`2100` 套；
- “其他”类别不可忽略，按普通住宅、非普通住宅建筑面积比分摊到两类税收核算；
- “列入/不列入”决定是否参与容积率核算；
- “允许扣除/不允许扣除”决定开发成本是否参与土地增值税扣除项目；
- 满意比例是每个房型的满意率，不是总和为 1 的需求份额，建模时应先归一化为相对偏好份额。

## 推荐模型链

### Q1 方案 I 核算

按房型计算：

```text
收入_i = 套数_i × 面积_i × 售价_i
建安成本_i = 套数_i × 面积_i × 单位开发成本_i
计容面积_i = 套数_i × 面积_i × 是否列入容积率
容积率 = Σ计容面积_i / 土地面积
```

土地增值税分普通住宅、非普通住宅两类核算。扣除项目建议包含：

```text
土地成本分摊 + 允许扣除开发成本 + 转让税金 + 开发费用 + 加计扣除
```

常用可复现口径：土地成本和转让税金按收入比例分摊；开发费用按土地成本与允许扣除开发成本之和的 10% 估算；加计扣除按 20% 估算。论文中必须说明这是清算测算口径，实际地方执行可有差异。

### Q2 方案 II 优化

先做 baseline：将满意比例归一化为偏好份额，并按总套数分配，再投影到房型上下界。该 baseline 通常匹配意愿很好，但可能违反容积率；用它作为“为什么需要优化”的证据。

主模型使用整数规划：

```text
决策变量：x_i = 第 i 种房型套数，整数
约束：l_i ≤ x_i ≤ u_i
      Σ x_i = 2100（与方案 I 规模可比，若论文另有设计可说明）
      Σ area_i × include_i × x_i ≤ 2.28 × 土地面积
目标：兼顾税前利润、满意度、供应结构偏差
```

供应结构偏差可写为：

```text
D(x)=Σ |x_i / 2100 - w_i|
```

用辅助变量线性化绝对值项，交给 `scipy.optimize.milp` 或其他 MILP 求解器。求解后必须再次用完整税后核算函数计算土地增值税、净利润和 ROI，不要只用线性目标值当最终利润。

### Q3 成功性与调整

投资回报率定义：

```text
ROI = (销售收入 - 总成本) / 总成本
```

与 25% 阈值比较。若方案 II 已达标，仍需做售价系数、建安成本系数的单因素敏感性分析；若不达标，给出售价上调、成本下降、房型结构调整的阈值或方向。

## 已验证的一组结果口径

在上述口径下，一次可复现求解得到：

- 方案 I：容积率约 `2.2752`，ROI 约 `33.24%`；
- 偏好比例 baseline：意愿匹配很好，但容积率约 `2.4456`，违反 2.28 上限；
- 方案 II：容积率约 `2.2799`，ROI 约 `34.16%`，达到 25% 成功阈值；
- 方案 II 示例套数：房型1--11 为 `50, 354, 50, 150, 100, 349, 450, 100, 50, 197, 250`。

这些数字依赖上述税收扣除估算口径。若使用不同开发费用、利息或土地分摊规则，必须重新冻结数字并同步论文。

## 代码与路径坑

- 若完整求解脚本放在 `支撑材料/代码/xxx.py`，项目根目录应取 `Path(__file__).resolve().parents[1]`，不是 `parents[2]`；否则输出会落到题目根目录而不是 `支撑材料/`，导致 zip 缺关键结果。
- Windows 中文路径可直接用 Python `Path` 处理；bash 命令中务必给路径加引号。
- 图表必须用显式中文字体 `font_manager.addfont('C:/Windows/Fonts/msyh.ttc')` 或黑体/宋体兜底，保存后抽查一张图确认中文不是方框。

## LaTeX 坑

- 参考文献若直接写 `[1] ... \\`，在普通段落中可能被 LaTeX 解析为可选参数并触发 `Illegal unit of measure (pt inserted)`。稳妥写法是使用：

```latex
\begin{enumerate}[label={[\arabic*]},leftmargin=2.5em]
\item 国务院. 中华人民共和国土地增值税暂行条例[Z]. 1993.
\end{enumerate}
```

- 自动生成论文时，金额单位建议在文本中直接写“亿元”，不要同时定义 `\yi` 命令又在数值后混用，避免命令漏定义或重复单位。
