# CUMCM 2017A CT 系统参数标定与返工踩坑记录

适用场景：处理 2017 年高教社杯 A 题（CT 系统参数标定与图像重建），或任何“由模板/附件标定扫描几何，再重建未知介质”的数学建模任务。

## 关键教训

- **不能只估计参数不进入重建链**：若论文写了标定角度、探测器间距、中心偏移等参数，后续附件 2/3/5 的重建必须显式使用这些参数；否则属于伪标定，用户可能要求返工。
- **优先建立模板回代验证**：先用附件 1/模板的已知几何或灰度图进行 Radon 正投影，与实测投影剖面对齐，确认标定参数能把模板重建回来，再处理未知附件。
- **必须有 baseline 对比**：交付前至少比较“未校正 baseline”和“标定校正后”的 RMSE、SSIM、投影相关系数等，证明标定有效。
- **逐投影 shift 校正要真实使用**：如果发现每条投影存在探测器方向平移，记录并在未知样本 sinogram 上逐投影应用 shift，而不是只在论文中描述。
- **交付前冻结数值**：将最终角度、步长、尺度、平移、误差指标写入 `results/frozen_numbers*.json` 或类似文件，论文表格/图形从该文件读取，避免论文和代码不一致。

## 推荐流程

1. **题意与数据读取**
   - 读取 `.docx` 题面、`.xls` 附件，确认每个附件对应模板、未知样本、投影序列还是坐标点。
   - 若路径含中文，避免把含中文路径直接作为 shell `workdir`；可在脚本中使用绝对路径，或从简单工作目录调用。

2. **模板 Radon 联合标定**
   - 构造模板图或模板几何掩膜。
   - 对候选参数做网格/连续优化：
     - 初始角 `theta0`
     - 角步长 `step`
     - 探测器尺度 `scale`
     - 投影中心/探测器平移 `shift[j]`
   - 目标函数建议使用归一化投影剖面相关系数、MSE/RMSE 或二者组合。

3. **模板回代验证**
   - 用标定参数重建模板。
   - 输出 baseline vs corrected 指标，例如：RMSE、SSIM、投影回代相关系数。
   - 若 corrected 没有显著优于 baseline，不要进入正式论文交付，继续检查角度方向、投影顺序、尺度和中心。

4. **未知附件重建**
   - 将同一套 `theta0/step/scale/shift` 用到附件 2、附件 3 等未知样本。
   - 采用 FBP/SART/TV 正则等方法重建；若用多方法，可把 FBP 作为 baseline，SART/TV 作为改进。
   - 对题目要求的坐标点取值、灰度矩阵、图像结果、误差/稳定性指标分别导出。

5. **论文与支撑材料**
   - 论文中明确写出：标定目标函数、参数估计结果、参数如何进入重建、模板验证结果、未知样本结果。
   - 支撑材料至少包含：主代码、结果表格、冻结参数 JSON、图像、验证脚本、README。
   - 最终压缩包核验：PDF 页数、关键文件存在、验证脚本通过、邮件发送成功。

## 交付质量门槛

- 模板标定后投影匹配相关系数应接近 1（具体阈值随数据而定，但必须给出）。
- 标定校正后的 RMSE/SSIM 等要优于 baseline，并在论文中量化。
- 附件 2/3/5 的处理不得重新随意拟合一套互不一致的几何参数，除非题目数据说明它们来自不同系统。
- 邮件或最终说明中要标注“以新版/返工版为准”，避免用户混淆旧版。
