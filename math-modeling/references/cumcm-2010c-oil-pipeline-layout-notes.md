# CUMCM 2010C 输油管布置案例笔记

## 适用场景

当题目要求在铁路、公路、管网入口等线性设施上选择接入点，并从两个或多个源点铺设管线/道路/电缆以最小化建设费用时可参考。典型特征：

- 两个源点位于同一侧，终点必须在线性设施上；
- 可选“不共用管线”或“先汇合再共用管线”；
- 不同区域有不同单位费用或附加费用；
- 不同管径/容量对应不同单位铺设费用。

## 坐标化建模模板

取铁路为 $x$ 轴，令 A 厂垂足 $C=(0,0)$，B 厂垂足 $D=(l,0)$：

- $A=(0,a)$，$B=(l,b)$；
- 城区/郊区分界线可设为 $x=c$；
- 车站为 $P=(p,0)$；
- 若允许共用管线，汇合点为 $J=(x,y)$，约束 $y\ge 0$。

### Baseline：不共用管线

\[
F_0(p)=G(A,P;w_A)+G(B,P;w_B)
\]

其中 $G(U,V;w)$ 是线段分段费用函数。无分区费用时可退化为加权 Weber 问题：

\[
F_0(p)=w_A\sqrt{p^2+a^2}+w_B\sqrt{(p-l)^2+b^2}.
\]

### 主模型：一处汇合点 + 共用管线

\[
F_1(x,y,p)=G(A,J;w_A)+G(B,J;w_B)+G(J,P;w_S),\quad y\ge0.
\]

最终比较 $\min F_0$ 与 $\min F_1$：若共用方案更低，则采用共用；否则保留 baseline。

## 分区费用函数

若线段 $U=(x_1,y_1)$ 到 $V=(x_2,y_2)$ 跨越分界线 $x=c$，交点参数为

\[
t=\frac{c-x_1}{x_2-x_1},\quad 0<t<1.
\]

总长 $L=|UV|$，则两侧长度分别为 $tL$ 与 $(1-t)L$，根据端点所在区域判断城区长度 $L_u$ 和郊区长度 $L_r$。线段费用：

\[
G(U,V;w)=wL_r+(w+q)L_u.
\]

其中 $q$ 是城区拆迁/补偿附加费用。

## 2010C 具体数值参考

题面参数：$a=5$，$b=8$，$c=15$，$l=20$（单位 km）。咨询公司估计城区附加费用为 21、24、20 万元/km，若按“甲级公司权重为每家乙级公司两倍”处理，则

\[
q=(2\times21+24+20)/4=21.5\text{ 万元/km}.
\]

在一次可复现求解中得到：

- 问题2（所有基础单位费用均 7.2 万元/km）：
  - 不共用 baseline：294.9931 万元；
  - 共用最优：$J=(3.4906,3.1669)$，$P=(3.4906,0)$，总费用 287.0572 万元；
  - 相对 baseline 节省 7.9359 万元，节省率 2.6902%。
- 问题3（A 专用 5.6、B 专用 6.0、共用 7.2 万元/km）：
  - 不共用 baseline：263.2988 万元；
  - 共用最优：$J=(4.3788,2.4041)$，$P=(4.3788,0)$，总费用 259.5648 万元；
  - 相对问题2最优节省 27.4923 万元，节省率 9.5773%。

这些数值可作为 sanity check，不要在新任务中硬编码；应以题面和用户要求重新计算。

## 代码实现注意事项

- Python 中不要把会在敏感性分析里变化的全局参数写成默认参数，例如 `def seg_cost(..., extra=urban_extra)`。默认参数在函数定义时绑定，会导致后续改变 `urban_extra` 时敏感性结果不变。改用：

```python
def seg_cost(P, Q, base, extra=None):
    if extra is None:
        extra = urban_extra
    ...
```

- 对连续非凸/分段目标，推荐先用 `scipy.optimize.differential_evolution` 做全局搜索，再用局部优化抛光；同时保留 baseline 作为可解释对照。
- 论文图中中文字体需显式设置，如 `SimHei`, `Microsoft YaHei`, `Noto Sans SC`，并使用 `plt.savefig(..., dpi=300, bbox_inches='tight')` + `plt.close()`。

## LaTeX 排版坑

在 `.tex` 正文中不要直接用 Markdown 反引号写代码文件名，尤其包含下划线时，例如 `` `frozen_numbers.json` `` 会触发 Missing `$` 或进入数学模式。应写成：

```tex
\texttt{frozen\_numbers.json}
```

## 论文表达要点

- 摘要必须同时给出“通用模型 + 问题2坐标/费用 + 问题3坐标/费用 + 节省率”。
- 问题一要写成通用判别流程，而不是只描述具体数值题。
- 问题二/三必须有不共用 baseline，否则无法说明共用方案是否真的更优。
- 敏感性分析优先扰动城区附加费用 $q$，观察费用和汇合点位置如何变化。
