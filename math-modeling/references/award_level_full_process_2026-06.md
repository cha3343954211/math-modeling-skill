# 获奖级数学建模全流程升级研究（2026-06）

> 目的：把 `math-modeling` 从“能完成题目”升级为“按获奖论文标准组织全过程”。本文件融合当前 skill 已有的 61 篇国赛获奖论文研究、G1-G6 门控、三层质量门控、论文质量检查清单，以及本机历次 CUMCM 案例复盘，形成可执行的获奖级工作规范。

## 1. 核心判断：获奖不是算法堆砌，而是证据链质量

高水平数模作品通常同时满足五件事：

1. **答题精准**：每个子问题都明确“题目要求输出什么 → 本文最终输出什么”，不跑偏。
2. **模型可信**：模型有现实解释、变量/参数/假设可追踪，复杂度与数据量匹配。
3. **数字可靠**：代码可复现，关键数字冻结，摘要/正文/图表数字一致。
4. **验证充分**：baseline 对比、题型对应检验、稳健性/敏感性至少覆盖多角度。
5. **论文会讲故事**：评委 3--10 分钟内能从摘要、图表、结论看懂贡献和可信度。

因此，获奖级流程的目标不是“找一个最炫模型”，而是搭建如下闭环：

```text
题目要求 → 题型识别 → 数据审计 → baseline → 候选模型PoC → 主模型
→ 可复现代码 → 结果冻结 → 验证/稳健性 → Claim-Evidence → 论文叙事 → 最终审计
```

## 2. 当前 skill 的已具备能力与新增差距

### 已具备能力

- G1-G6 门控：审题、方法 PoC、代码审查、结果冻结、论文写作、审计。
- L1/L2/L3 三层质量门控：建模合理性、求解正确性、论文质量。
- 61 篇获奖论文规律：小问闭环、摘要数字化、baseline、验证稳健、图表解释。
- 支撑材料标准目录：`questN/codes/figures/outputs` 与 `papper/`。
- 大量 CUMCM 案例踩坑记录：路径、LaTeX、图表、装箱伪可行、公式错用等。

### 仍需强化的获奖级差距

1. **竞赛策略层不足**：需要开题选题、72 小时资源分配、放弃/降级策略。
2. **评委视角不够显式**：需要把“评分点 → 论文位置 → 证据文件”绑定。
3. **创新包装不系统**：需要区分真实创新、组合创新、验证创新、应用创新，避免空喊创新。
4. **结果叙事模板不足**：需要强制每个关键结果采用“数值 + 对比 + 含义 + 稳健性”。
5. **最终打磨不够硬**：需要提交前进行摘要、图表、页数、公式、引用、附件压缩包的独立审计。

## 3. 获奖级全流程：S0-G7

### S0 赛题预检与选题策略

开题后先做 30--90 分钟快速评估：

| 维度 | 检查问题 | 决策含义 |
|---|---|---|
| 数据可用性 | 附件是否完整、字段是否可读、样本量是否足够 | 数据坏/缺附件则风险高 |
| 输出明确性 | 题目要求方案、预测、排序、公式还是解释 | 输出越明确越容易闭环 |
| 团队匹配 | 是否掌握对应模型、代码和论文叙事 | 选熟悉题优先 |
| 题型风险 | 是否需要大量外部数据、复杂证明或不可控仿真 | 风险过高需降级 |
| 可验证性 | 是否能做 baseline、误差/约束/稳定性检验 | 无法验证则难高分 |

选题原则：**熟悉 + 可验证 + 能写清楚 > 看起来新奇**。

### G1 题目解析与评分点对齐

必须生成 `rubric_alignment.md/json` 或等价记录：

```text
子问题 | 题目要求 | 评分关注点 | 本文输出 | 证据文件 | 论文位置
Q1 | ... | 模型合理/结果准确/图表清楚 | ... | ... | 5.1, 6.1
```

每问同时完成：12 类题型标注、输入输出表、依赖图、符号初表、难点与风险。

### G2 模型路线与创新定位

候选模型池至少包含 baseline + 主模型。主模型要明确获奖价值来自哪类创新：

| 创新类型 | 合格写法 | 不合格写法 |
|---|---|---|
| 问题抽象创新 | 把现实问题拆成可求解的多阶段模型，并解释阶段传递 | “本文模型具有创新性” |
| 组合模型创新 | 预测→优化、评价→调度、机理→仿真，数据流清楚 | 随意堆多个算法 |
| 约束/指标创新 | 新增现实约束、业务指标或可执行评价指标 | 加无意义权重 |
| 验证创新 | 小规模精确解、反事实、情景/稳健性、回代校验 | 只说结果合理 |
| 可视化/解释创新 | 图表直观揭示机制、异常和决策含义 | 堆漂亮但无证据图 |

每个候选主模型必须用真实数据小切片 PoC，失败即记录 `[REJECTED]`。

### G3 正式代码与实验管理

正式代码必须做到：

- 一键运行或有清晰运行命令；
- 固定随机种子；
- 输出 `run_summary.json`；
- 保存中间表、最终表、图表和日志；
- 优化题必须输出约束残差；预测题必须输出训练/测试指标；评价题必须输出权重与排名稳定性。

推荐 `run_summary.json` 字段：

```json
{
  "question_id": "Q1",
  "run_command": "python quest1/codes/q1_main.py",
  "random_seed": 42,
  "input_files": [],
  "output_files": [],
  "metrics": {},
  "constraints_check": {},
  "exit_code": 0
}
```

### G4 结果冻结与证据包

结果冻结不只是写 `frozen_numbers.json`，还要形成每问材料包：

```text
qx_solution_package_for_writer.md
├── 本问题目要求与最终输出
├── 最终采用模型与放弃模型
├── 核心公式/变量说明
├── 关键结果表与图
├── frozen_numbers 键列表
├── baseline 对比
├── 稳健性/敏感性结果
└── 论文写作注意事项
```

没有材料包，不进入论文定稿。

### G5 论文叙事与 Claim-Evidence

每个关键主张都要有证据来源：

```text
claim_id | 主张 | 数字/图表 | frozen_numbers键 | 证据文件 | 论文位置 | 风险边界
```

关键结果采用固定表达：

```text
数值：模型得到 ___；
对比：相比 baseline/现状 ___；
含义：说明 ___；
稳健性：在 ___ 扰动/检验下仍 ___。
```

### G6 评委视角审计

最终提交前模拟评委快速阅读：

1. **只看摘要**：是否知道每问方法和关键数字？
2. **翻模型章节**：是否有变量、公式、约束、算法步骤？
3. **看图表**：图表是否支撑结论，而不是装饰？
4. **看验证**：是否证明模型优于 baseline 或至少可信？
5. **看结论**：是否逐问回答题目，没有新增无证据主张？

### G7 提交包终审

交付前必须核验：PDF 可打开、页数达标、摘要 1 页、目录 1 页、图表/公式未溢出、参考文献不少于 5 条、支撑材料 zip 包含代码/结果/图表/冻结数字/readme。

## 4. 题型获奖加分点

| 题型 | 必做 baseline | 高分验证 | 论文亮点 |
|---|---|---|---|
| 预测 | 上一期/移动平均/线性回归 | 时间切分、RMSE/MAE/MAPE、残差、外推风险 | 解释误差来源和趋势可信区间 |
| 优化 | 现状/均分/贪心/小规模穷举 | 约束残差、小规模最优、敏感性 | 可执行方案表 + 改善幅度 |
| 评价 | 等权/单指标排序 | 权重一致性、指标方向、排名稳定性 | 指标体系来源清楚，排名解释具体 |
| 机理 | 简化模型/经验公式 | 量纲、边界、参数扰动、回代 | 参数有物理意义，仿真图解释机制 |
| 聚类分类 | KMeans/多数类/简单阈值 | 轮廓系数/F1/混淆矩阵/稳定性 | 簇画像/分类规则能解释现实 |
| 仿真 | 无干预/简单规则 | 重复实验、置信区间、情景对比 | 情景建议可落地 |
| 混合题 | 每阶段 baseline | 误差传播、跨问一致性 | 数据流图展示前问如何支撑后问 |

## 5. 72 小时获奖级时间分配

```text
0-3h：选题预检、题目解析、附件审计、竞赛策略
3-8h：资料/IMA/论文/官方文档检索，候选模型与 baseline
8-20h：Q1 baseline + PoC + 主模型 + 初步结果 + 小结
20-34h：Q2/Q3 顺序闭环，必要时回溯前问
34-46h：验证、稳健性、图表、冻结数字、材料包
46-60h：论文主干成文，Claim-Evidence 映射
60-68h：摘要、结论、图表解释、格式排版、参考文献
68-72h：G6/G7 终审、PDF/zip 核验、补漏提交
```

若时间不足，优先级为：

```text
结果正确 > 摘要有数字 > 小问闭环 > baseline/验证 > 图表解释 > 格式细节 > 高级算法
```

## 6. 获奖级最低交付标准

正式建模交付若目标为“获奖级”，最低必须满足：

1. 每问有 baseline 或明确无需 baseline；
2. 每问有主模型数学表达和求解算法；
3. 每问有结果表/图 + 2--4 句解释；
4. 每问至少一种题型对应验证；全篇至少一处敏感性/稳健性；
5. 摘要逐问写方法和具体数字；
6. 关键数字全部来自冻结结果；
7. PDF 正文一般不少于 15 页，摘要与目录各 1 页；
8. 支撑材料可复现，包含代码、图表、最终结果、冻结数字和 readme。

## 7. 反面红线

出现以下任一问题，不得宣称“获奖级完成”：

- 摘要无数字或遗漏子问题；
- 模型不能直接产出题目要求的结果；
- 无 baseline 却宣称模型优秀；
- 代码没跑通或结果不可复现；
- 论文数字与输出文件不一致；
- 优化方案违反硬约束；
- 图表无数据来源或趋势与文字矛盾；
- 引用、数据、实验、结论有虚构风险；
- 正式论文像简报，缺少推导、求解、验证和评价。

## 8. 写入主 Skill 的执行原则

本研究应转化为主 `SKILL.md` 的硬规则：

- 启用“获奖级模式”时，必须执行 S0-G7；
- 增加 `rubric_alignment`、`solution_package_for_writer`、`claim_evidence_map`；
- 以评委视角审计摘要、模型、图表、验证和提交包；
- 把创新写成“可验证的改进”，不要写空泛创新；
- 任何不足 15 页正文、无冻结数字、无验证的正式论文默认 L3 不通过。
