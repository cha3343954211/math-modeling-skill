# 论文质量检查清单

## 问题现象

用户反馈“论文后半部分写的不完善”“论文页数太少”“格式不规范”——初稿常见问题是正文只有 8--10 页、摘要像普通报告、缺少目录/页眉页码/承诺书占位、章节深度不足、图表解释太短、附录和支撑材料说明过粗。

## 根本原因

- 只把计算结果塞进论文，没有按竞赛论文结构展开“问题分析—模型建立—求解—检验—评价”的完整叙述。
- 使用了普通 article/ctexart 简易模板，未显式设置摘要页、目录页、页眉页脚、标题层级、图表标题、三线表、参考文献和附录。
- 论文生成脚本追求简洁，未设置最低正文篇幅、图表解释字数和排版检查门槛。
- 误把“能说明模型”当成“竞赛可提交论文”：数学建模正式论文通常应给评委足够推导、过程、表格、检验和局限分析。

## 格式与篇幅硬门槛（国赛/CUMCM风格）

正式交付前必须执行以下检查：

```
□ PDF 总页数建议 15--25 页；开放题/评价题最低不应少于 14 页。若只有 8--10 页，默认判定为 L3 待补。
□ 正文结构必须包含：题目、摘要、关键词、目录、问题重述、问题分析、模型假设、符号说明、数据来源与预处理、模型建立与求解、模型检验/灵敏度分析、模型评价/改进/推广、参考文献、附录。
□ 摘要单独成块，300--800 字，包含背景、每个子问题的方法、关键数值结果、稳健性结论；关键词 3--5 个；**摘要只能占 1 页**，不得跨页。若放不下，压缩背景、套话和非关键细节，保留方法、关键数字和结论。
□ 目录必须单独成页且**只能占 1 页**。若目录放不开，调整目录字号、行距、段前段后和标题显示深度（如只显示到二级标题），不要让目录占 2 页。
□ 正文小四/12pt，中文宋体或等价 ctex 字体，1.25--1.5 倍行距；页边距约 2.5cm；页码底部居中；页眉可写论文题名缩写。
□ 一级标题、二级标题层级清楚，避免全篇只有少量大段文字。
□ 图题在图下，表题在表上；所有图表必须编号、单位明确，并在正文引用和解释。
□ 表格优先三线表（booktabs），宽表用 longtable/resizebox，不要溢出页面。
□ 公式居中编号；公式中的符号必须在符号说明中定义；**公式必须显示完全**，不得被裁切、越过页边距、与编号重叠或异常跨页。长公式用 `aligned`/`split`/`multline`/`cases` 拆行处理。
□ 参考文献不少于 5 篇，格式统一，优先 GB/T 7714 风格；网页/数据库要标明 DB/OL、EB/OL。
□ 附录包含核心代码、支撑材料目录、复现说明；大段代码可放支撑材料，论文附录只放关键入口与说明。
```

### LaTeX 规范配置建议

正式中文数学建模论文优先使用 `ctexart` + `geometry` + `fancyhdr` + `booktabs` + `longtable` + `caption` + `setspace`。推荐设置：

```latex
\documentclass[UTF8,a4paper,12pt]{ctexart}
\usepackage{geometry}
\geometry{left=2.5cm,right=2.5cm,top=2.5cm,bottom=2.5cm}
\usepackage{setspace}
\setstretch{1.35}
\usepackage{fancyhdr}
\pagestyle{fancy}
\fancyhf{}
\fancyhead[C]{\small 论文题名缩写}
\fancyfoot[C]{\thepage}
\setlength{\headheight}{14pt}
\usepackage{amsmath,amssymb,booktabs,longtable,array,graphicx,float,caption,hyperref}
\captionsetup[table]{position=top}
\captionsetup[figure]{position=bottom}
```

### 章节深度最低要求

| 章节 | 最低要求 | 推荐要求 |
|------|----------|----------|
| 摘要 | 300字+关键数字 | 500--800字，逐问题给方法和结果 |
| 问题重述 | 300字 | 背景+任务分解+本文侧面定义 |
| 问题分析 | 600字 | 每个子问题单独分析，说明方法选择理由 |
| 数据预处理 | 500字+数据表 | 数据来源、口径、缺失、标准化、审计表 |
| 模型建立与求解 | 3000字 | 分问题写公式、算法步骤、结果表、图表解释 |
| 模型检验 | 800字 | baseline、敏感性、随机/残差/约束检验 |
| 模型评价 | 800字 | 优缺点和改进逐条展开，不写套话 |
| 附录 | 复现入口 | 代码入口、文件结构、运行命令、冻结数字说明 |

### 质量红线

出现下列情况，不能直接交付，必须回炉修改：

```
□ PDF 少于 14 页且用户未要求“短报告”
□ 没有目录或页码
□ 摘要没有具体数字
□ 任一图表没有正文解释
□ 参考文献少于 5 条或格式混乱
□ 附录没有说明代码与支撑材料如何复现
□ 论文中的数字不是从 frozen_numbers.json 或最终结果表读取
```

## 质量检查清单

### 问题三（预测/反事实/趋势类）必须包含

```
□ 模型训练与测试的评价指标（如 RMSE、MAE、MAPE、R²，按题型选择）
□ 拟合/预测效果图或反事实对比图
□ 残差或误差分析（均值、标准差、最大误差等，按题型选择）
□ 预测/反事实结果与历史数据的对比验证
□ 结果可靠性论证：趋势期合理性、外推风险、异常年份解释
```

### 灵敏度分析必须包含

```
□ 确定2个以上关键参数或关键权重
□ 每个参数的扰动范围和步长
□ 扰动结果表格或图形
□ 参数灵敏度对比分析
□ 结论：哪个参数更敏感、模型是否稳定
```

### 模型评价必须包含

```
□ 优点：每条100字以上，说明原理和应用价值
□ 缺点：每条100字以上，分析具体影响和原因
□ 改进：每条100字以上，说明实施方法和预期效果
□ 优点数量 >= 缺点数量
```

## 输出前自检

写完论文后，对照以下问题检查：

1. PDF 是否达到 15 页左右？若不足，是否补充了模型推导、数据表、图表解释、检验和附录？
2. 是否有目录、页码、页眉/页脚、规范标题层级？
3. 摘要是否有关键数字、排名、误差/稳健性结果？
4. 每个子问题是否都有“问题要求→模型→求解→结果→解释→检验”？
5. 灵敏度分析是否分析了2个以上参数/权重？
6. 模型评价每条是否有详细解释？
7. 所有表格是否有分析文字？所有图表是否有解读？
8. 参考文献与附录是否完整、格式统一？
