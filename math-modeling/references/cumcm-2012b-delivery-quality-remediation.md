# CUMCM 2012B 太阳能小屋交付质量补救记录

## 触发信号

用户指出初版论文只有约 12 页，且格式不像正式数学建模论文；这类反馈说明交付物虽然有结果，但未通过 L3 论文质量门控。

## 可复用教训

1. **页数必须实测**：交付前用 `pdfinfo 论文.pdf` 或等价工具读取页数，不能凭观感判断。若总页数 12--14 页，目录/附录扣除后正文通常不足 15 页，默认需要扩充。
2. **短报告≠正式论文**：只罗列模型、结果表和结论会显得像“结果报告”；正式建模论文需要展开题目对齐、变量与约束、数据审计、算法步骤、结果解释、baseline 对比、敏感性/稳健性分析和模型评价。
3. **返工扩充优先补证据链**：不要靠空话凑页数。优先补：数据清洗表、候选组件/参数表、约束核验表、每问阵列/方案结果表、baseline 与主模型差异、参数误差传播、求解流程伪代码、图表正文解释。
4. **补发时要明确以最新版为准**：若已经发送过不合格附件，应重新生成 PDF 和支撑材料 zip，重新邮件发送，并在最终回复中说明最新版页数、编译错误数、文件路径和邮件发送结果。

## 推荐补救流程

```text
1. 读取 PDF 页数和 LaTeX 编译日志；
2. 对照 L3 清单找缺项：正文页数、目录、页眉页码、三线表、图表解释、baseline、敏感性；
3. 扩写论文主体，而不是只扩摘要/附录；
4. XeLaTeX 编译 2--3 遍，确认错误数为 0；
5. 重新打包支撑材料，打开 zip 清单核验论文、代码、结果表、readme 是否在内；
6. 邮件补发 PDF + zip，并标注“优化版/最新版”。
```

## 2012B 特定注意

太阳能小屋类优化题要把“组件铺设、逆变器选配、建筑外表面可用面积、串并联约束、投资收益/回收期”写成完整闭环。不能只给最终收益，应解释组件筛选、朝向/面积约束、容量匹配、逆变器数量与输入范围、以及参数扰动对经济性的影响。
