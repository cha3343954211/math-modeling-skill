# CUMCM 2012A 葡萄酒评价建模交付记录

本记录提炼一次完整完成 2012A「葡萄酒的评价」任务时的可复用经验，供后续类似“感官评分 + 理化指标 + 芳香物质 + 论文 PDF + 邮件交付”任务参考。

## 题型与推荐流程

- **Q1 评分可信性**：同一批样品由两组评委评分，优先用配对检验而不是独立样本检验。
  - 解析每个酒样评分块，对 10 名评委分别累计分项总分。
  - 使用配对 `t` 检验 / Wilcoxon 检验判断两组均分差异。
  - 使用样品内标准差、变异系数、Cronbach alpha、评委间 Spearman 秩相关辅助判断可信度。
- **Q2 葡萄分级**：不要只按葡萄酒评分排序；应把“成酒质量”和“原料客观指标”结合。
  - 酿酒葡萄理化指标 + 葡萄芳香物质：中位数填补、Z-score 标准化、PCA 得葡萄指标分。
  - 综合分示例：`0.55 * 可信酒质评分 + 0.45 * 葡萄PCA分`，再用 KMeans 或分位数分级。
  - 对权重做敏感性分析，报告排名 Spearman 稳定性。
- **Q3 葡萄—葡萄酒指标联系**：高维小样本下避免大量两两相关堆砌。
  - 分别对葡萄指标矩阵和葡萄酒指标矩阵做 PCA，比较主成分相关。
  - 用 PLS 从葡萄指标预测葡萄酒主成分，交叉验证 `R²/RMSE` 作为联系强度证据。
- **Q4 理化指标评价酒质**：比较 grape / wine / combined 三类特征集。
  - baseline：线性回归。
  - 主模型：RidgeCV、PLS、RandomForest 等，必须用交叉验证而非训练集拟合误差。
  - 结论通常应写成“可辅助评价/预筛选，但不能完全替代专家品评”，除非外部验证效果非常强。

## Excel 解析注意

- 附件1评分表结构不统一：红葡萄酒表常以“酒样品N”作为块首；白葡萄酒表可能样品编号在第 1 列、样品名在第 2/3 列，且每块行数不完全一样。
- 稳健解析方式：扫描每行所有单元格，只要匹配 `样品\s*(\d+)` 就作为新评分块；当前块到下一块之间，取每行最后 10 个数值作为 10 位评委分项评分并累加。
- 附件2存在平行测定列：同一指标的 1/2/3 列应先求均值，再进入建模。
- 附件3芳香物质空值多表示未检出；可按 0 处理，但论文中要声明假设和局限。

## LaTeX 自动生成常见坑

- Python raw string / 模板替换中不要把 `\n` 写进正文。LaTeX 会把它当未定义控制序列；应写真实换行或 `\\`。
- 中文标点紧跟行内数学时，必须闭合 `$...$`：
  - 错：`$R^2=0.553，说明...`
  - 对：`$R^2=0.553$，说明...`
  - 错：`$p=0.0439，白葡萄酒$p=0.0071`
  - 对：`$p=0.0439$，白葡萄酒$p=0.0071$`
- 编译时不要只看 PDF 是否生成；要 grep 日志中的 `^!`、`Undefined control sequence`、`Missing $ inserted`。这些错误即使生成 PDF，也可能导致中文字消失或公式错乱。
- 若最终正文页数略低于 15 页，应主动补充：数据审计表、评分解析说明、PCA载荷解释、更多分级表、模型对比解释和附录说明。
