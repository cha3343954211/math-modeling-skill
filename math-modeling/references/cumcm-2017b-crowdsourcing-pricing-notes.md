# CUMCM 2017B “拍照赚钱”的任务定价案例

## 适用题型

移动众包 / 空间供需 / 任务定价 / 完成概率预测 / 打包发布优化。

当题目给出历史任务位置、价格、完成情况，以及会员/服务者位置、能力或信誉信息时，可复用本案例路线。

## 数据结构

典型字段：

- 历史任务：任务编号、GPS 纬度、GPS 经度、任务标价、任务执行情况；
- 会员信息：会员编号、会员位置、预订任务限额、预订开始时间、信誉值；
- 新项目任务：任务编号、GPS 纬度、GPS 经度。

## 建模主线

把任务完成建模为“价格 + 空间供需 + 任务竞争”共同决定的完成概率问题：

\[
Y_i\in\{0,1\},\qquad P_i=\Pr(Y_i=1|p_i,X_i)
\]

其中 `p_i` 是价格，`X_i` 是空间供需特征。

推荐完整流程：

```text
数据审计
→ Haversine 距离矩阵
→ 多半径空间供需特征
→ 原定价规律回归 + 完成概率模型
→ 目标完成率反推单任务价格
→ 空间平滑与 0.5 元阶梯化
→ DBSCAN 密集任务打包
→ 新项目迁移定价与效果评价
```

## 空间特征工程

对每个任务 `i` 和半径 `r ∈ {1,2,3,5,10} km`，计算：

\[
N_i(r)=\sum_j I(d_{ij}\le r)
\]

附近会员数量；

\[
C_i(r)=\sum_{j:d_{ij}\le r} credit_j,
\qquad
L_i(r)=\sum_{j:d_{ij}\le r} limit_j
\]

附近会员信誉供给与预订限额供给；

\[
T_i(r)=\sum_{k\ne i}I(d_{ik}\le r)
\]

附近任务竞争数量；

\[
Q_i(r)=\frac{L_i(r)}{T_i(r)+1},
\qquad
M_i(r)=\frac{N_i(r)}{T_i(r)+1}
\]

按预订能力和会员数量定义的供需比。

距离采用 Haversine：

\[
d_{ij}=2R\arcsin\sqrt{\sin^2\frac{\varphi_i-\varphi_j}{2}+\cos\varphi_i\cos\varphi_j\sin^2\frac{\lambda_i-\lambda_j}{2}}
\]

## 问题一：原定价规律与未完成原因

### Baseline

1. 只用价格解释完成：`logit(done ~ price)`；
2. 只用空间供需比解释完成：`logit(done ~ Q_i)`；
3. 原价格回归：`price ~ lat + lon + supply + demand + distance`。

### 主模型

- 逻辑回归：保留解释性和反推价格能力；
- 随机森林 / GBDT：捕捉非线性空间关系，用 AUC、LogLoss、Brier 验证；
- 置换重要性或 SHAP：解释关键因素。

完成概率逻辑模型：

\[
P_i=\sigma(\alpha_0+\alpha_p p_i+\bm\alpha^T X_i)
\]

若 `α_p > 0`，价格对完成概率有正向边际作用。

### 常见结论表达

- 完成任务均价通常高于未完成任务，说明价格有正向作用；
- 价格相关性通常不强，说明完成情况还受空间供需影响；
- 经纬度、附近任务密度、供需比、最近会员距离等通常是重要特征；
- 原定价若集中在较窄区间，会导致高风险区域补偿不足。

## 问题二：目标完成率反推定价

给定目标完成概率 `τ`，求满足概率约束的最低价格：

\[
\min p_i \quad s.t.\quad P_i(p_i)\ge \tau,
\quad p_{min}\le p_i\le p_{max}
\]

逻辑模型可解析反解：

\[
p_i^*=\frac{\log\frac{\tau}{1-\tau}-\alpha_0-\bm\alpha^T X_i}{\alpha_p}
\]

然后：

1. 截断到价格区间，例如 `[65,90]`；
2. 按历史定价习惯 0.5 元阶梯化；
3. 对邻近任务做空间平滑，避免同一区域价格突变：

\[
p_i^{smooth}=(1-\rho)p_i^*+\rho\frac{1}{|\mathcal N_i|}\sum_{k\in\mathcal N_i}p_k^*
\]

### 比较指标

- 总预算、平均价格；
- 平均预测完成率；
- 高风险任务数量，例如 `P_i < 0.6`；
- 单位预算期望完成任务数；
- 空间价格平滑性。

## 问题三：任务打包模型

### 候选包生成

使用 DBSCAN 在任务坐标上识别密集簇：

- 距离度量：Haversine；
- `eps ≈ 1 km`；
- `min_samples = 2`；
- 包大小建议限制 `2 ≤ |G| ≤ 3`，避免包过大降低接受意愿。

簇内用贪心近邻形成任务包，并约束最大包内距离，例如：

\[
\max_{i,j\in G}d_{ij}\le 1.5\text{ km}
\]

### 打包价格

设包内单任务价格总和为 `Σp_i`，包内平均两两距离为 `\bar d_G`，按紧凑度给予折扣：

\[
p_G=\sum_{i\in G}p_i\left(1-\eta\frac{1}{1+\bar d_G}\right)
\]

`η` 可取 0.08--0.15，并做敏感性分析。

### 打包完成概率

无历史打包数据时，可采用可解释修正：

\[
P_G=clip\left(\frac1{|G|}\sum_{i\in G}P_i
+\theta_1\frac{1}{1+\bar d_G}
-\theta_2(|G|-1),0.05,0.98\right)
\]

其中第二项表示路径节约收益，第三项表示包大小负担。

### 接受准则

注意：包完成意味着包内多个任务同时完成，所以效率应使用“单位成本期望完成任务数”，而不是简单比较 `P_G/p_G` 与单任务均值。

\[
E_{single}=\frac{\sum_{i\in G}P_i}{\sum_{i\in G}p_i},
\qquad
E_{bundle}=\frac{|G|P_G}{p_G}
\]

接受打包条件：

\[
E_{bundle}>E_{single},\qquad P_G\ge0.9\bar P_G
\]

第二个条件防止为省钱而让包完成概率明显降低。

## 问题四：新项目迁移定价

对新任务构造同样空间特征，使用历史任务训练好的完成概率模型定价。若发现新项目任务密度显著高于历史项目，单任务价格可能大量触及上限且预测完成率仍低，此时论文必须解释：

- 新项目空间竞争更强，单纯提价不足以解决；
- 需要打包、分批发布、提高局部上限或调整发布时间；
- 不能把价格上限截断后的低预测概率伪装成“高完成率方案”。

输出材料建议：

- `附件一新定价方案.xlsx/csv`；
- `附件一打包方案.xlsx/csv`；
- `附件三新项目单任务定价方案.xlsx/csv`；
- `附件三新项目打包建议.xlsx/csv`；
- 模型指标表、特征重要性表、数据审计表、summary/frozen numbers。

## 实现注意事项

- Excel 读取后统一重命名列，会员位置字符串要拆分为纬度和经度；
- `task_cnt_rkm` 对历史任务自身计数时要减去 1；
- 逻辑回归做反推价格时，若使用 `StandardScaler`，要把系数还原到原始尺度后再解析反解；
- GBDT/RF 只能做预测和验证，反推价格优先使用逻辑模型或网格搜索；
- 图表中文字体需显式加载 Windows 字体并抽查，所有图表 `savefig + close`；
- 若终端 `workdir` 因中文路径被工具拦截，可在简单 ASCII 临时目录编译/运行中转，但所有最终产物必须复制回用户指定题目目录的标准 `支撑材料/` 树。

## 论文写作要点

摘要必须包含具体数字：原完成率、原均价、模型 AUC、新方案均价/预算变化、预测完成率变化、打包覆盖任务数和节约率等。

每问闭环：

```text
题目要求 → 输入字段 → baseline → 主模型公式 → 算法步骤 → 输出表/图 → 结果解释 → 验证/局限
```

特别要说明打包模型没有历史打包标签，其完成概率是基于路径节约的可解释仿真，属于实施效果评价而非真实观测完成率。
