# CUMCM 2012D 机器人避障问题建模记录

## 适用场景

当题目是机器人/车辆在二维平面、多障碍物环境中的避障路径规划，且要求路径由直线段和圆弧段组成、转弯半径有下限、与障碍物保持安全距离时，可参考本记录。

## 题目要点

- 场景：`800 x 800` 平面区域。
- 障碍物：12 个几何障碍，包括正方形、长方形、三角形、平行四边形和圆形。
- 安全约束：机器人路径与任一障碍物最近距离至少 `10` 个单位。
- 运动约束：路径由直线段和圆弧段组成，圆弧与直线相切；不能用折线瞬时转弯；每段圆弧半径至少 `10`。
- 输出：每段直线/圆弧的起点、终点；圆弧圆心；总距离与总时间。

## 推荐建模路线

### 1. 配置空间 / 安全膨胀

把每个障碍物按安全距离 `d_s=10` 做 Minkowski 膨胀：

```text
P_i^+(d_s) = {x : dist(x, P_i) <= d_s}
C_free = [0,800]x[0,800] \ union_i int(P_i^+(d_s))
```

这样“路径距障碍物至少 10”就转化为“不进入膨胀障碍物内部”。正方形、长方形、三角形、平行四边形用多边形表示；圆形可用高分辨率圆盘/多边形近似，但论文中要说明近似精度。

### 2. 可视图 + Dijkstra baseline

对起点、终点、膨胀障碍物边界顶点建可视图：

- 节点：起点、终点、膨胀障碍物外边界顶点；
- 边：两节点连线完全位于可行空间；
- 权重：欧氏距离；
- 求解：Dijkstra 最短路。

该结果是安全折线最短路 baseline，不满足“不能折线转弯”，但能给出全局绕障拓扑和理论参考距离。

### 3. 圆角化成可执行路径

对折线中连续三点 `P_{k-1}, P_k, P_{k+1}`，用相切圆弧替代尖角：

```text
u = (P_{k-1}-P_k)/||P_{k-1}-P_k||
v = (P_{k+1}-P_k)/||P_{k+1}-P_k||
phi = arccos(u·v)
d = r tan(phi/2)
Q1 = P_k + d u
Q2 = P_k + d v
b = (u+v)/||u+v||
O_k = P_k + r/sin(phi/2) b
arc_length = r theta
```

若相邻线段太短，实际半径应截断为局部可行值，例如：

```text
r_k = min(r, 0.45 * min(L1, L2) / tan(phi/2))
```

为了避免圆角化后弧线向障碍物一侧偏移导致安全裕度不足，可用更大的骨架膨胀半径（如 `20`）先生成折线骨架，再圆角化；论文中需明确这是为圆弧平滑预留安全裕度。

### 4. 时间模型

直线段长度：`l = ||P_end-P_start||`；圆弧长度：`l = r theta`。

若题面速度公式抽取完整，可按题面公式；若 Word 公式对象只显示为 `[pic]`，必须回查原始文件/官方题面，不能无说明地硬写。若暂时无法恢复公式，参数化处理并在论文中明示假设：

```text
T = sum(line_length / v_line) + sum(arc_length / v_turn(r))
```

本次 2012D 交付中采用了 `v_line=5`、`v_turn(r)=min(5, sqrt(10r))` 的参数化口径。以后复用时应优先核对官方 PDF/图片中的速度常数；如果常数不同，只需改代码中的速度参数并重新生成冻结数字、论文摘要和结果表。

## 论文写作要点

- 摘要必须给出各条路径的具体距离和时间。
- 解释 baseline 与主模型区别：可视图折线最短路是理论参考；圆角化路径才满足机器人转弯约束。
- 每张路径图要说明灰色障碍物、膨胀边界、路径线的含义。
- 模型检验至少包括：可视图 baseline 对比、安全性回代、转弯半径约束检查、半径敏感性分析。
- 若速度公式来自假设或题面公式存在提取缺失，必须在“模型局限/假设”中写明，并把速度常数保存在代码参数和 `frozen_numbers.json` 中。

## 常见坑

1. **只求折线最短路不够**：题目要求不能折线转弯，必须输出圆弧段。
2. **圆角化可能破坏安全距离**：圆弧相对折线会偏移；应留额外骨架安全裕度或对圆弧采样做距离回代。
3. **Word 公式提取为 `[pic]`**：不要把缺失公式当作已知；优先找官方 PDF/图片或用参数化假设并明示。
4. **组合路线顺序固定**：`O-A-B-C-O` 是指定访问顺序时，分段求解后拼接；若题目允许重排，再引入 TSP。
5. **论文数字传播**：速度参数、圆角半径或安全裕度改变后，必须重新运行代码、更新 `frozen_numbers.json`、摘要、正文表格和结论。
