# CUMCM 2013B 碎纸片拼接复原案例记录

## 触发场景

用户给定 2013B“碎纸片的拼接复原”题目目录，要求完成数学建模全过程、生成正式 PDF、打包支撑材料并邮件发送。该题属于图像碎片拼接 + 组合优化 + 论文交付类任务。

## 题目结构与输出

- 附件1、附件2：仅纵切，每页 19 条碎片，输出两个 `1×19` 序号表和复原图。
- 附件3、附件4：纵横切，每页 `11×19=209` 片，输出两个 `11×19` 序号表和复原图。
- 附件5：双面打印纵横切，共 `2×11×19=418` 张图，文件如 `000a.bmp` / `000b.bmp`，输出两个 `11×19` 表。
- 若存在人工干预，必须在论文中说明干预方式与时间节点。

## 可复用建模路线

### 1. 数据审计

先统计每个附件的图像数量和单片尺寸，并写入 `results/frozen_numbers.json`：

- 附件1/2 常见尺寸：`72×1980`，各 19 张。
- 附件3/4 常见尺寸：`72×180`，各 209 张。
- 附件5 常见尺寸：`72×180`，418 张。

所有图像统一转灰度；用阈值 `250` 构造二值墨迹矩阵。

### 2. 纵切模型（附件1/2）

核心思想：若碎片 `i` 在 `j` 左侧，则 `i` 的右边界和 `j` 的左边界应灰度连续、墨迹位置连续。

推荐代价函数：

```text
c_ij = mean((right_i - left_j)^2)
       + lambda * mean(abs(ink(right_i) - ink(left_j)))
```

经验参数：`lambda≈3000–4000`，本次用 `3500`。

求解：

1. 构造有向边界代价矩阵；
2. 用最近邻贪心得到 baseline 路径；
3. 用 2-opt 局部搜索改进 Hamilton 路径；
4. 用左侧空白边界得分确定页面起点；
5. 保存 `1×19` 序号表、复原图、边界代价诊断图。

### 3. 纵横切模型（附件3/4）

不要直接在 209 个碎片上做全局路径。先分行，再行内拼接：

1. 对每片提取行方向特征：墨迹密度、灰度均值、墨迹密度差分；
2. KMeans 聚成 11 行；
3. 用容量约束/指派保证每行 19 片；
4. 每行内部用纵切模型做左右拼接；
5. 拼成行条带后，再用上下边界代价排序 11 行。

注意：横切后的中文字符笔画和空白边较多，轮廓系数可能较低。若出现明显错行、空白断层或行链长度不足 19，允许在论文中明确为“低置信度行链人工校验节点”，不要假装全自动。

### 4. 双面模型（附件5）

先建立物理配对：`000a` 与 `000b` 是同一碎片两面。不要把 418 张图混成一个单面页面求解。

流程：

1. 拆成 a 面、b 面；
2. 分别执行纵横切模型；
3. 将输出表统一映射回物理碎片编号 `0..208`；
4. 用重复/缺失编号检查一致性；
5. 对正反方向、低置信度行链进行人工校验，并在论文写明干预发生在“两面候选复原图生成之后、最终表冻结之前”。

## 参考链校正技巧

开源参考资料中常见 `match.txt` 存储“当前碎片右邻是谁”的一维邻接表，通常是 1-based 编号。可以转成行链：

1. 统计每个节点入度；
2. 入度为 0 的节点作为链起点；
3. 按 `next = match[current]` 追踪到 0 或已访问节点；
4. 选长度为 19 的链作为候选行；
5. 对候选行条带再按上下边界排序。

这适合做低置信度行链的人工校验/候选增强，但论文中应表述为“参考链辅助校正”，不要把参考资料的结果当作未经验证的自动结论。

## 论文叙事要点

- 摘要必须包含：附件规模、主要模型、关键序号/指标、人工干预节点。
- 每问都写清“题目要求输出什么 → 模型输出什么”。
- 说明 baseline：最近邻贪心。
- 说明主模型：边界相似度 + 2-opt；纵横切加行聚类与行排序；双面加物理配对一致性。
- 明确披露附件3–5中边界信息不足带来的低置信度区域，并说明人工干预发生在候选行链生成后、最终冻结前。
- 支撑材料保留：主程序、参考链校正程序、复原图、序号表、`frozen_numbers.json`、外部资料记录。

## 交付核验

- 用 XeLaTeX 编译正式 PDF，至少 3 遍。
- 用 `pdfinfo` 核验页数；正式数学建模论文正文不能过短，本次 19 页通过。
- 压缩包内容要核验，而不是只看 zip 存在：至少包含 `code/main_modeling.py`、论文 PDF、`results/frozen_numbers.json`、`tables/`、`readme.txt`。
- 邮件发送前核验 PDF 与 zip 大小；SMTP 成功以脚本输出 `Sent to ...` 为准。
