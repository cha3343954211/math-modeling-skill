# 国赛优秀论文归档 → 研究汇报 PPT 工作流

适用场景：用户要求“研究某目录下所有国赛/数学建模优秀论文，生成研究汇报 PPT，并邮件发送”。本文件记录本次成功流程的可复用步骤，避免未来重复摸索。

## 1. 批量识别与抽取

- 先用文件搜索确认 PDF 数量与年份/题型分布。
- 对本地 PDF 优先使用 `pymupdf/fitz` 抽取文本；逐篇保存到 `texts/`，同时生成指标表。
- 输出目录建议：`<WORKSPACE>/<主题>/`，至少包含：

```text
<主题>/
├── texts/                  # PDF 抽取文本
├── figures/                # 汇报图表
├── paper_metrics.csv       # 原始指标
├── paper_metrics_scored.csv# 加权/评分指标
├── summary.json            # 汇总统计
├── 研究报告.md
├── <研究汇报>.pptx
└── <材料包>.zip
```

## 2. 建议抽取指标

每篇论文记录：

- 文件路径、年份、题型字母、页数、字符数。
- 摘要文本与“摘要是否含数字”。
- 公式/数字/标题近似数量。
- 方法关键词计数：优化/规划、评价/决策、预测/回归、统计/数据分析、机理/仿真、图论/网络。
- 质量要素关键词：baseline/对比、误差/残差、灵敏度/稳健性、图表/可视化、附录/代码。

注意：PDF 文本抽取会受双栏、扫描页、公式和页眉页脚影响；自动指标只能作为结构化线索，最终结论要结合人工建模经验解释。

## 3. 汇报 PPT 内容骨架

推荐 10–12 页：

1. 封面：样本量、年份、题型、高分规律。
2. 样本与方法：数据来源、抽取方法、指标维度、局限。
3. 一句话结论：高分论文赢在可信证据链。
4. 题型—方法栈：A/B/C/D/E 对应建模方法。
5. 高分小问闭环：题目要求→输入数据→baseline→主模型→求解→结果解释→验证。
6. 摘要数字化：方法 + 数字 + 结论。
7. 验证与稳健性：按预测/优化/评价/仿真分类。
8. 图表策略：图表服务结论，每张图必须解释。
9. 72 小时实战流程。
10. 三层质量门控。
11. Skill/知识沉淀。
12. 备赛行动清单。

## 4. PPT QA

- 若没有 LibreOffice/soffice，不能做真正的图片级渲染时，不要跳过 QA。
- 至少做结构 QA：
  - `python-pptx` 打开文件并确认页数、尺寸、shape 数、图片存在。
  - `markitdown` 抽取文本，检查顺序和缺失。
  - 扫描占位符：`TODO|TBD|FIXME|lorem|placeholder|Click to add|待补|占位`。
  - 检查长文本框、过多形状、空白页、缺失媒体关系。
- 重要交付建议再让子代理做一次只读 QA；没有渲染时在最终说明里标注“已做结构 QA，未做图片级视觉 QA”。

## 5. 邮件交付

- 邮件发送前必须验证 PPT 和 zip 均存在且 zip 内含关键文件。
- 使用 `email-file-delivery` 的 `send_smtp_attachment.py` 或薄 wrapper，凭证从环境变量读取，不写入脚本。
- 邮件正文说明：样本来源、附件内容、主要结论。
- SMTP 返回 `Sent to ...` 后再向用户报告已发送。

## 6. 同步更新 math-modeling skill

完成此类研究后，将可复用结论写入 `references/`，并在 `SKILL.md` 支持文件表中加入一行指针。不要把完整 session 日志塞进主 Skill；主 Skill 保持规则，细节放引用文件。
