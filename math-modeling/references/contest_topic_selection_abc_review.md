# 校赛 ABC 题选题快速评估记录（2026-06）

## 适用场景

当用户给出一个竞赛题目文件夹，要求比较 A/B/C 题难易程度、求解可行性和选题建议时，优先采用“题面 + 附件 + 已有支撑材料”三源交叉评估，而不是只凭题目类型泛泛判断。

## 快速评估流程

1. **目录预检**
   - 查看根目录下 A/B/C 子目录、题面 PDF、附件目录、支撑材料目录、压缩包是否存在。
   - 若已有 `支撑材料/`，读取 `readme.txt`、`results/frozen_numbers.json`、`contracts/*.json`、论文 PDF 页数等，用于判断完成度和风险。

2. **题面提取**
   - 用 PDF 文本提取工具读取每道题的：背景、子问题数量、附件说明、给定参数、硬约束、输出要求。
   - 重点摘取“问题一/二/三/四”和附件说明，不必全文复述。

3. **附件与数据充分性判断**
   - A 类：参数是否全给定、是否无需外部数据。
   - B 类：是否有图像附件、Excel 坐标、噪声参数；图像读点/反演是否会成为不稳定源。
   - C 类：是否有坐标、区域、约束、设备参数；是否需要自行补充公开资料或人为设定。

4. **已有成果质量核验（如存在）**
   - 读 `frozen_numbers.json` 抽取关键结果，而不是从论文正文猜。
   - 用 `pdfinfo` 或等价工具核验论文页数；数学建模正式论文若明显低于 15 页，应标记 L3 风险。
   - 对比 readme 的质量门控说明和实际文件，避免只看压缩包存在。

5. **评分维度**
   - 难度：数学/算法复杂度、数据处理难度、验证难度。
   - 可行性：给定数据充分性、模型闭环、代码可复现、论文可解释性。
   - 风险：参数拍脑袋、图像读点误差、反演不稳定、题目过大导致叙事空泛。
   - 出彩点：背景新颖、图形展示、数学推导、工程应用价值。

## 本次校赛 ABC 题经验

- **A 题：校园共享单车优化**
  - 类型：ODE 连续动力学 + 差分预测 + 调度优化 + 场景/灵敏度分析。
  - 优势：参数完整、题意闭环自然、结果直观，最适合稳妥参赛。
  - 风险：创新感略弱，应在调度目标、回代验证、场景稳健性上补强。

- **B 题：球面方位投影航线识别**
  - 类型：球面几何投影 + 带噪反投影 + TSP/多目标优化。
  - 优势：数学味强、图形效果好，做得好很出彩。
  - 风险：图像读点、仿射误差逆变换、盲区城市反推和投影公式正确性是高风险点；论文页数不足时需补大量推导与误差验证。

- **C 题：无人机低空航线与空域调度**
  - 类型：网格化评价 + 路径规划 + 多机协同调度 + TOPSIS 效能评估。
  - 优势：低空经济背景新、工程展示感强、论文容易写丰满。
  - 风险：题目宏大，若参数补充和简化假设解释不足，容易“大而虚”。

## 输出建议格式

建议先给一句明确排序，例如：

```text
推荐优先级：A题 > C题 > B题
```

然后按每题写：

```text
1. 题目特点
2. 求解路线
3. 优点
4. 缺点/风险
5. 选题结论
```

最后按不同目标给建议：

- 稳妥拿奖：选数据完整、闭环强、风险低的题。
- 冲创新：选数学或工程展示感强但风险高的题。
- 时间紧：避免图像反演、开放参数过多、验证难以落地的题。
