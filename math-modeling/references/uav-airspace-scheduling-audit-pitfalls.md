# 无人机低空航线优化与空域调度：审计修正版经验

适用场景：低空无人机空域网格化、航线覆盖、多机调度、空域冲突消解、综合效能评价等数学建模题。

## 关键经验

1. **题面空间约束必须进入网格生成**
   - 若题面给出禁飞区、安全缓冲区、河道/建筑/道路边界，不能只在论文中描述，必须在网格筛选或约束矩阵中实际扣除。
   - 网格数量、有效面积、覆盖率均应由扣除后的有效网格集合统计。

2. **单机航线结果要区分“全域工作量”和“单次任务能力”**
   - 全域巡检总航程/总时间不能直接等同于单架无人机一次任务。
   - 必须计算单次续航约束下的最大任务包面积/长度、所需架次或批次数，并在论文中明确说明。

3. **多机调度必须绑定任务包与空间单元**
   - 只给每架无人机的开始/结束时间不足以证明覆盖率或冲突消解。
   - 每个任务包至少应包含：`task_id`、`grid_id` 或覆盖网格集合、任务类型、持续时间、无人机编号、起止时间、高度层。
   - 覆盖率应从“任务包—网格覆盖矩阵”统计，而不是手动设定或封顶。

4. **冲突检测不能只看时间重叠**
   - 至少检查同网格同时间重叠、水平邻近重叠、垂直高度间隔、容量超限事件。
   - 若采用高度分层，应报告各任务类型高度层和最小垂直间隔，并说明其满足安全阈值。

5. **综合评价指标必须来自前面模型结果**
   - TOPSIS/AHP 等评价不能把优化方案预设为全指标最优。
   - 安全性、效率、成本、覆盖、稳定性等指标应由 Q1--Q3 的真实输出映射；若需归一化或经验系数，应说明来源和敏感性。

6. **审计发现问题后必须全链路传播修正**
   - 修改模型代码后，重新运行模型、更新 `frozen_numbers.json`、重生表格/图、重写论文相关段落、重新编译 PDF、重新打包。
   - 若已经发送过初版，应补发修正版邮件并在正文说明“以最新附件为准”。

## 建议输出表

- 网格评价表：`grid_id, x, y, type, score, is_valid`
- 单机航线对比表：baseline 与优化航线的距离、时间、转弯数、节省率、续航可行性
- 单次任务能力表：任务类型、单次可覆盖面积/长度、所需架次、是否满足续航
- 任务包表：`task_id, task_type, grid_id, duration_min, assigned_drone, start_min, end_min, altitude_m`
- 冲突审计表：冲突类型、baseline 冲突数、优化后冲突数
- 综合评价表：各方案指标值、正/逆向属性、权重、TOPSIS 贴近度、排名
- 敏感性表：关键参数或权重扰动下的核心结果与排名

## 论文表述边界

- 启发式排程应称为"可行优化方案/启发式优化"，不要称为"全局最优"，除非有严格 MILP/穷举/最优性证明。
- 高度分层消冲应说明适用条件：通信可靠、飞控可执行、气象满足、监管允许。
- 视觉巡检宽度、传感器覆盖半径等参数若题面未给，应作为假设并做敏感性分析。

## 代码级易错坑点（2026-06 C2 实战新增）

### 传感器宽度单位陷阱
`coverage_analysis(area_name, sensor_width_m=12)` 的参数单位是**米**，函数内部 `/1000` 转为 km。
```python
# ✅ 正确：120 米
river = coverage_analysis('黄河防汛巡检', 120)  # 120m → 0.12 km

# ❌ 错误：传 0.120 被当作 0.12 米 → 0.00012 km → 8334 条带
river = coverage_analysis('黄河防汛巡检', 0.120)
```
**症状**：黄河巡检航程暴增到十几万 km，架次数万——立即检查传感器宽度参数。

### 冲突检测半径过小导致零冲突
网格间距 0.5 km = 500 m。若冲突检测水平半径取 `hsep_m * 20 = 300 m`，相邻格对永不触发冲突。
```python
# ❌ 错误：300m 永远碰不到 500m 邻居格
if d < PARAM['hsep_m'] * 20 and abs(h_diff) < vsep:

# ✅ 正确：至少 2000m 覆盖 ~16 个邻居格
if d < 2000 and abs(h_diff) < PARAM['vsep_m']:
```
**症状**：Q3 不管 baseline 还是优化都显示 0 冲突——立即检查检测半径。

### Q4 综合评价指标需基于 Q3 实际输出后校准
不能在 Q3 结果生成前预设 Q4 指标值。必须读取 Q3 的 `base_conf`、`opt_conf`、`coverage_rate` 后动态计算：
```python
# ✅：Q3 跑完后才构造 Q4 指标
random_safety = max(0.30, 1.0 - base_conf['total_conflicts'] / (len(base_sched) * 8.0))
opt_safety = max(0.60, 1.0 - opt_conf['total_conflicts'] / (len(sched) * 4.0))
```
**症状**：优化方案 TOPSIS 反而低于传统方案或排名第 3——Q4 指标值与 Q3 输出不协调。

### LaTeX 生成中的 Python f-string 反斜杠限制
Python f-string 表达式内不能含反斜杠，不能用 `f"\text{{...}}"` 这类写法。
```python
# ❌：f-string 语法错误
tex = f"... L_{{\text{{opt}}}} = ..."

# ✅：预先计算所有含反斜杠的值，用普通变量插入
opt_label = "L_{\\text{opt}}"
tex = f"... {opt_label} = ..."
```
或直接使用 `write_file` 写入完整 `.tex` 而非 f-string 拼接。

### 压缩包自包含导致 176 MB
```python
# ❌：zip 路径在支撑材料/package/ 内，glob 会包含自身
for f in root.rglob('*'):
    z.write(f)

# ✅：排除旧 zip 或输出到支撑材料目录之外
if 'package' in f.parts or f.name.endswith('.zip'):
    continue
```
**症状**：几 MB 的实际文件打包成 176 MB 的 zip——旧 zip 被嵌套压缩。

### Windows 文件锁定
打包脚本运行时若 `zipfile.ZipFile` 未正确关闭，或终端/explorer 仍持有句柄，`zip_path.unlink()` 抛出 `PermissionError`。
**解法**：用新文件名（如 `C2_支撑材料.zip` 替代 `C2_论文与支撑材料.zip`）绕开锁定。
