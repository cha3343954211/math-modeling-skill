# CUMCM 论文格式与篇幅补救记录

## 触发信号
用户质疑“数学建模论文为什么就 10 页、格式不规范”。这类反馈说明 L3 论文质量门控不能只检查摘要数字、图表编号和三层审计，还必须检查正式竞赛论文的篇幅、版式和可读性。

## 可复用判断
若任务是“完成数学建模全过程 / 最终 PDF / 可提交论文”，除非用户明确要求短报告，否则 8--10 页 PDF 默认偏短，应判定为 L3 待补。正式中文数学建模论文通常应落在 15--25 页左右，开放评价题最低也应尽量达到 14 页以上。

## 补救动作清单
1. 先说明问题：短 PDF 往往是“结果报告”，不是完整竞赛论文。
2. 对照格式要求扩充：
   - 标题页/摘要/关键词；
   - 目录；
   - 页眉、页脚、页码；
   - 问题重述、问题分析、模型假设、符号说明；
   - 数据来源、指标体系、数据审计；
   - 模型建立、公式推导、算法步骤；
   - 结果表、权重表、贡献表、年度表；
   - baseline 对比、灵敏度/稳健性、误差来源；
   - 模型评价、局限、改进推广；
   - 参考文献和附录/支撑材料说明。
3. LaTeX 排版建议：
   - `ctexart` + `geometry` + `fancyhdr` + `setspace` + `booktabs` + `longtable` + `caption`；
   - 表题在上、图题在下；
   - `xelatex` 编译至少 2--3 遍；
   - 用 `pdfinfo` 检查页数；
   - 检查 log 中 `undefined reference/citation`、明显 `Overfull`。
4. 支撑材料中保留格式改进审计，说明页数、排版、章节、表格、引用和附录做了哪些修复。
5. 重新压缩和交付前再次验证 PDF 页数与附件大小。

## 技术细节
- 生成 LaTeX 表格行时，注意每行必须以 `\\` 结束。Python 字符串里应写成 `" \\\\"` 或用原始字符串谨慎处理；单个反斜杠会导致 LaTeX 表格报 `Missing \\cr inserted`。
- 若参考文献英文长行导致 `Overfull \\hbox`，可在参考文献 `enumerate` 环境中局部加入 `\\sloppy`，结束后 `\\fussy`。

## 目录压缩与“讲明白建模求解”补救

若用户反馈“目录太长/目录压缩到一页以内”或“论文没说明白建模求解和结果”，按以下流程修复：

1. **目录先压缩**：优先让目录只保留一级章节；若 `\setcounter{tocdepth}{0}` 在 ctex/既有 `.toc` 流程中仍输出多级条目，直接改为手写一页紧凑目录（标题“目录” + 一级章节 + `\dotfill` 页码），避免目录占两页影响观感。
2. **不要只扩字数**：用户要的是“说明白”，不是泛泛加长。必须补清楚一条可追溯计算链：
   `PDF/附件数据识别 → 结构化数据文件 → 距离/指标矩阵 → 数学模型 → 求解算法 → 结果表/图 → 时间/方案/排名等题目答案 → 约束检验`。
3. **在模型建立与求解处补操作性步骤**：例如路径优化题要写清楚“建图—求路径—换算时间—检验约束”；评价题写清“指标正逆向—归一化—权重—综合得分—排名稳定性”；预测题写清“划分—训练—测试误差—未来预测”。
4. **结果解释要回答‘数字是什么意思’**：每个关键结果至少补 2--3 句解释：相对 baseline 变化、主要原因、对题目决策的含义、约束是否满足。不要只罗列“结果为 X”。
5. **重新编译和验证**：XeLaTeX 至少 2 遍；用 `pdfinfo` 核验页数；必要时用 PDF 文本抽取或目视检查确认目录确实只有一页；重新打包并标注“优化版/以本封为准”。

## 质量门槛
交付前至少确认：
- PDF 页数 ≥ 14（推荐 15--25）；
- 摘要含关键数字；
- 目录、页码、页眉/页脚存在；
- 图表均有编号、标题和正文解释；
- 参考文献 ≥ 5 条且格式统一；
- 附录说明代码入口、冻结数字和复现方法。
