# CUMCM 2017C 颜色与物质浓度辨识案例

## 适用场景

当题目给出比色法/颜色传感数据，要求判断颜色数据质量、建立颜色到浓度的预测模型、分析样本量和颜色维度影响时，可参考本案例。

## 推荐建模路线

1. **输入资产预检**：确认题面 `.docx` 与 Excel 附件可读，先在题目目录下建立标准 `支撑材料/`，不要把产物放到临时 sibling 目录。
2. **数据审计**：读取 Excel 时先用 `header=None` 查看原始表格结构，再决定表头、浓度行、空白/水样本和错位数据的解析口径。
3. **白板/零浓度校正**：若存在“水”样本或零浓度基准，用其作为白板基准构造相对色差特征，可降低底色和光照的系统偏差。
4. **问题一质量评价**：不要只看单一相关系数；组合以下指标更稳健：
   - 单调性；
   - 相邻浓度可分性；
   - 重复测量稳定性；
   - 留一预测误差；
   - 综合质量分与排名。
5. **问题二浓度预测**：小样本优先使用可解释、可交叉验证的模型。有效路线是五维颜色特征（RGB/HSV 或 RGBSH 等）+ 二次多项式特征 + Ridge 正则化，并用 LOO 交叉验证比较 linear / polynomial / ridge / lasso 等候选模型。
6. **问题三样本量与维度影响**：
   - 按样本比例降采样，比较 LOO RMSE 或 MAE；
   - 枚举颜色维度组合，比较误差、特征重要性和 Spearman 相关性；
   - 对噪声扰动做敏感性分析。
7. **冻结数字**：生成 `results/frozen_numbers.json`，论文摘要和正文关键数字只从冻结文件与最终结果表引用。
8. **交付核验**：编译 PDF 后用 `pdfinfo` 核验页数；打包支撑材料后用 `zipfile` 检查关键文件是否包含，再发送邮件。

## 代码实现坑点

- **中文路径 workdir 阻断**：Hermes `terminal` 的 `workdir` 可能拒绝包含中文字符的路径（如 `年`）。做法是使用安全工作目录（如 Hermes agent 根目录），在 shell 命令中再 `cd 'E:/.../中文路径/...' && ...`。
- **小样本抽样越界**：按比例抽样时不要写 `n=max(8, int(len(df)*frac))` 后直接抽样；当样本量小于 8 会越界。应写成：

```python
n = min(len(df), max(8, int(len(df) * frac)))
```

- **Excel 表格错位**：颜色数据表可能存在“水”样本、数值浓度行、重复测量和局部错位。正式建模前先打印原始单元格窗口，写稳健解析逻辑，再输出清洗后的表格供人工审计。
- **中文图表**：Windows/Hermes 环境中 Matplotlib 只设 `SimHei` 可能仍出方框。优先显式加载 `C:/Windows/Fonts/msyh.ttc` 或 `simhei.ttf`，并用视觉工具或人工抽查至少一张关键图。
- **LaTeX 生成**：动态 Python 字符串生成中文 `.tex` 容易因反斜杠/三引号转义导致 `SyntaxError`；复杂论文可直接用 `write_file` 写完整 `.tex`，再 XeLaTeX 编译三遍。

## 论文表达要点

- 明确写出“题目要求 → 模型输出”的映射：数据质量评价、颜色--浓度预测、样本量/维度影响。
- 对问题一综合质量分解释其现实含义：哪些物质颜色响应稳定、哪些物质不适合建立浓度模型。
- 对问题二报告最优模型名称、交叉验证 RMSE/MAE/R²、真实-预测对比图和残差分析。
- 对问题三解释样本量减少带来的误差变化、最佳颜色维度组合及其可能原因。
- 交付邮件正文写清附件包含论文 PDF 和完整支撑材料压缩包，并列出 2--3 个关键结果，便于用户快速核对。
