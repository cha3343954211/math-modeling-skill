# CUMCM 2015C “月上柳梢头”案例记录

## 适用题型

- 开放式天文/物理解释题：题目无数据附件，需要把文学或自然语言描述转成可计算的物理/天文阈值。
- 需要输出“日期 + 时间窗口”，而不是单点时间或泛泛解释。
- 需要多地区比较，核心差异来自经纬度、本地太阳时、天体高度角变化。

## 可复用建模路线

### S0 输入资产预检

- 原题目录只有 `CUMCM-2015-problem C-Chinese.docx`，无 Excel/CSV 附件。
- 可用 `pandoc <docx> -t plain -o 支撑材料/data/problem_statement.txt` 提取题面。
- 按标准结构原地创建 `支撑材料/papper/`、`code/`、`tables/`、`results/figures/`、`contracts/`、`data/`。

### Q1 定义与验证

- “月上柳梢头”：用月亮真高度角阈值表示。可取 `8°–15°` 作为主模型范围；低于约 8° 容易受地平遮挡，高于约 15° 则弱化“柳梢头”的低空意象。
- “黄昏后”：用太阳真高度角表示。可取 `-12°–-6°`，对应民用暮光结束后到航海暮光前。
- 明月意象过滤：月面照明比例建议 `>=0.80`。
- 验证：用 2015 年北京农历正月十五附近回代。一次复现得到北京 2015-03-05 窗口 18:40–19:08，代表时刻 18:54，太阳高度角约 -9.28°、月亮高度角约 10.74°、照明比例约 0.999。

### Q2/Q3 2016 城市判定

- 用 `lunardate` 得到农历正月十五对应公历日期，再对正月十五前后 3 天扫描。
- 用 `skyfield` + `de421.bsp` 计算太阳、月亮相对观测点的地平坐标。
- 对每个城市本地 15:00 到次日 01:00 逐分钟扫描：

```text
满足条件 iff
  -12° <= sun_alt <= -6°
  8° <= moon_alt <= 15°
  moon_illumination >= 0.80
```

- 合并连续满足分钟，输出开始时间、结束时间、代表时间、持续分钟。代表时间可取连续窗口中位时刻。
- 2016 年一次复现结论：北京 2016-02-22 18:25–18:56；哈尔滨、上海、广州、昆明、成都、乌鲁木齐均可发生，代表时间大致从哈尔滨 17:57 到乌鲁木齐 20:37。东西部钟表时间差异主要由经度和本地太阳时造成。

### Baseline 与验证

- Baseline：只检查“农历正月十五 + 太阳高度角接近 -6° 时月亮高度是否在 8°–15°”。
- 主模型优势：baseline 只看单点，可能漏掉稍晚进入交集的城市；逐分钟扫描能给完整窗口和持续时间。
- 敏感性：将月亮高度区间或黄昏太阳高度区间整体平移 `±2°`，检查可发生城市数和平均持续时间。一次复现中城市能否发生结论稳定，持续时间约从 29.1 分钟降到 19.7 分钟量级。

## 代码实现要点

- `skyfield` 矢量化计算比逐分钟逐点循环快很多：一次构造时间数组，`loc.at(t).observe(...).apparent().altaz()` 直接返回数组。
- 保存图表时仍遵守用户偏好：`plt.savefig(..., dpi=300, bbox_inches='tight')` 后 `plt.close()`，不要 `plt.show()`。
- 中文图表字体显式加载 Windows 字体，例如 `C:/Windows/Fonts/msyh.ttc`，生成后抽查至少一张图，避免中文方框。
- Windows/Hermes 的 `terminal(workdir=...)` 可能因中文路径被安全策略拦截。遇到中文赛题目录时，不要把中文路径放进 `workdir`；用默认 workdir 并在命令中完整引用目标路径，或用 Python `Path` 读写。

## LaTeX 与交付注意

- 论文要说明“题目无附件；使用公开星历和城市经纬度进行确定性计算”，不要把估计阈值写成题目给定数据。
- 正文需要把文学意象与数学阈值一一对应：月相照明比例对应“明月”，月亮高度角对应“柳梢头”，太阳高度角对应“黄昏后”。
- 交付前用 `pdfinfo 论文.pdf` 核验页数。若正文偏短，可补充城市差异解释、诗句意象一致性分析、代码复现与数字一致性检验，而不是只调字号或堆附录。
- 用补丁工具编辑 `.tex` 中含 `\ref`、`\texttt` 等反斜杠命令后，必须抽读被编辑行或重新编译检查；若反斜杠被误转义，立即修复再交付。
