# 数学建模论文详细写作与严谨性增强指南

## 目标

本指南用于解决“论文不够详细、逻辑不够严谨、结果分析不充分”的问题。最终论文必须清晰讲述完整数学建模求解过程，而不是只展示模型名称和最终答案。

## 一、论文叙事主线

正式论文应围绕以下证据链展开：

```text
题目要求
→ 问题抽象：把现实语言转成数学对象
→ 数据与假设：说明数据口径、假设必要性
→ 变量符号：定义决策变量、参数、单位
→ 模型构建：目标函数/评价函数/预测函数/约束来源
→ 模型求解：算法步骤、参数设置、实现方式
→ 结果输出：表格、图形、关键数值
→ 结果分析：数值含义、对比、趋势、原因
→ 模型检验：误差、约束、baseline、稳健性
→ 评价改进：优缺点、推广、局限
```

写作时不要让章节彼此割裂。每个子问题都要说明它与前一问、后一问的关系，例如“问题一得到的指标权重将作为问题二优化模型的参数”。

## 二、每个子问题的标准闭环

每个子问题建议按以下 10 个小节组织：

1. **题目要求与输出对齐**
   - 本问要求求什么？
   - 题目希望输出数值、排序、预测、方案还是解释？
   - 本文模型最终输出什么，如何对应题目要求？

2. **输入数据与预处理**
   - 使用哪些数据表/字段？
   - 样本量、时间范围、单位是什么？
   - 缺失、异常、重复、单位换算如何处理？

3. **baseline 模型**
   - 给出最简单可解释的基线方法。
   - 说明 baseline 的作用：判断主模型是否真的有提升。

4. **主模型选择理由**
   - 本问属于预测/优化/评价/分类/仿真/机理哪一类？
   - 为什么选择该模型而不是其他模型？
   - 主模型相比 baseline 的预期优势是什么？

5. **变量、参数与符号定义**
   - 变量、参数、指标、单位统一。
   - 公式中出现的符号必须进入符号说明表。

6. **模型建立与公式推导**
   - 公式要来自题意，不要突然出现。
   - 目标函数/损失函数/评价函数要解释现实含义。
   - 约束条件要解释为什么存在、来自题目哪句话或现实限制。

7. **求解算法与实现步骤**
   - 给算法步骤或伪代码。
   - 说明 Python/MATLAB 求解库、参数设置、随机种子。
   - 若用启发式算法，说明终止条件和收敛判断。

8. **结果呈现**
   - 用表格呈现精确值，用图形呈现趋势、关系和对比。
   - 关键结果给单位、保留位数和来源文件。

9. **结果分析**
   - 解释最大/最小值、排名、趋势、拐点、异常点。
   - 与 baseline、常识、题目约束或历史数据对比。
   - 说明结果对实际问题的意义。

10. **本问小结与传递关系**
    - 用 3--5 句话总结本问答案。
    - 若结果供下一问使用，明确传递哪些参数或结论。

## 三、结果分析写作模板

结果分析不能写成“由图可知结果较好”。建议使用：

```text
由表X可知，方案A的综合得分为0.873，较baseline方案提高12.4%。
该结果说明在同时考虑成本、效率和稳定性时，方案A能够取得更均衡的表现。
从指标贡献看，效率指标的权重最高，因此方案A在效率维度的优势是其排名第一的主要原因。
同时，方案C虽然成本最低，但稳定性指标明显偏低，导致综合得分下降。
这表明本题不能仅按单一成本指标决策，而需要采用多指标综合评价。
```

至少覆盖四类信息：

- **数值本身**：是多少，单位是什么；
- **对比关系**：比谁高/低，提升多少；
- **原因解释**：为什么出现这个结果；
- **实际意义**：对题目决策有什么启示。

## 四、严谨性与正确性检查

### 1. 题意与模型一致性

```text
□ 每个子问题都有“题目要求输出 → 模型输出”的映射。
□ 变量和目标函数能直接回答题目，而不是回答了相似但不同的问题。
□ 若做了简化，说明简化不改变核心结论。
```

### 2. 数据正确性

```text
□ 原始数据只读保存。
□ 清洗步骤可复现。
□ 单位换算有记录。
□ 训练/测试或时间划分合理。
□ 没有数据泄漏。
```

### 3. 模型正确性

```text
□ 假设在模型中被实际使用。
□ 目标函数方向正确：最大化/最小化不反。
□ 约束条件完整且单位一致。
□ 权重、概率、比例等变量落在合理范围。
□ 优化结果满足约束，预测结果符合常识边界。
```

### 4. 代码正确性

```text
□ 关键函数有小样例验证。
□ 固定随机种子。
□ 输出最终结果表、图片、日志和 frozen_numbers.json。
□ 图表保存使用 plt.savefig(...) + plt.close()，不使用 plt.show()。
```

### 5. 论文正确性

```text
□ 摘要、正文、结论中的数字一致。
□ 论文数字都能追溯到 frozen_numbers.json 或最终结果表。
□ 图表标题、单位、编号完整。
□ 每个图表正文至少解释 2--4 句。
□ 参考文献不少于 5 篇且格式统一。
```

## 五、增强求解过程的推荐做法

### 预测类

- baseline：上一期、均值、移动平均、线性回归。
- 主模型：ARIMA、灰色预测、随机森林、XGBoost、组合预测。
- 检验：MAE、RMSE、MAPE、残差图、训练测试划分、滚动预测。
- 论文必须说明外推风险和预测区间。

### 优化类

- baseline：当前方案、贪心、均分、小规模穷举。
- 主模型：线性/整数规划、动态规划、网络流、启发式算法。
- 检验：约束残差、目标函数对比、可行性检查、关键参数敏感性。
- 论文必须说明约束来自哪里，以及方案是否可执行。

### 评价类

- baseline：等权评分、单指标排序。
- 主模型：熵权-TOPSIS、AHP-TOPSIS、组合赋权、PCA。
- 检验：权重合理性、排名稳定性、等权对比、指标方向检查。
- 论文必须解释每个指标的现实含义和正/负向属性。

### 分类/聚类类

- baseline：多数类、简单阈值、KMeans 初版。
- 主模型：随机森林、SVM、XGBoost、DBSCAN、层次聚类。
- 检验：混淆矩阵、F1、轮廓系数、簇稳定性、可解释性。
- 论文必须解释每一类/每一簇的实际含义。

## 六、页数不足时优先补充内容

当正文不足 15 页时，优先补充以下内容，而不是用空话凑字：

1. 题目要求拆解表；
2. 数据字段与单位说明表；
3. 数据预处理审计表；
4. baseline 模型与主模型对比；
5. 公式推导和约束来源说明；
6. 算法步骤或伪代码；
7. 关键中间结果表；
8. 每张图表的 2--4 句解释；
9. 误差、残差、约束满足或排名稳定性检验；
10. 敏感性分析表和图；
11. 模型优缺点与改进对应表；
12. 附录中的复现说明和支撑材料目录。

## 七、最终审计输出格式

交付前在最终回复或审计文件中给出：

```text
L1 建模合理性：通过/待补
- 子问题映射：...
- 假设与变量：...
- baseline：...

L2 求解正确性：通过/待补
- 数据审计：...
- 代码复现：...
- 结果冻结：...
- 检验方法：...

L3 论文质量：通过/待补
- 正文页数：...
- 摘要数字：...
- 图表解释：...
- 参考文献/附录：...
```
